# 设计模式

程序设计是思维具体化的一种方式，是思考如何解决问题的过程，设计模式是在解决问题的过程中，一些良好思路的经验集成，最早讲设计模式，人们总会提到 Gof 的著作，它最早将经典的 23 种模式集合在一起，对后期学习编程设计的人们起到了莫大的影响。后来设计模式一词被广泛的应用到各种经验集成，甚至还有反模式(AntiPattern)，反模式教导如何避开一些常规且似是而非的程序设计思维。
